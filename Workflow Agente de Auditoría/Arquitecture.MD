\# 🏗️ Arquitectura del Agente – Auditoría de Accesibilidad



\## Componentes

\- Input Layer → recibe URL, APK, sitio o repo.

\- Validation Layer → ejecuta validaciones WCAG, ONTI, BCRA.

\- Output Layer → genera informe MD y JSON.



\## Harness

\- WCAG Harness → pruebas técnicas.

\- ONTI Harness → validaciones legales nacionales.

\- BCRA Harness → requisitos regulatorios financieros.



\## Flujo de datos

1\. Usuario provee recurso.

2\. Harness ejecuta validaciones.

3\. Resultados se consolidan en checklist.

4\. Se generan salidas MD y JSON.



\## Portabilidad

\- Claude Code

\- Gemini CLI

\- Copilot / Cursor
